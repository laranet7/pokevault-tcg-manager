# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in PokeVault TCG, please do not open a public GitHub issue with the details.

Instead, please report it privately to the maintainer.

When reporting a vulnerability, include:

- A clear description of the issue.
- Steps to reproduce it.
- Affected version or commit, if known.
- Possible impact.
- Suggested fix, if available.

Please do not exploit the vulnerability, access other users' data, or disclose the issue publicly before it has been reviewed.

## Scope

Security issues may include:

- Authentication bypass.
- Authorization issues.
- Exposure of API keys, secrets or tokens.
- Access to another user's data.
- SQL injection.
- Cross-site scripting.
- Unsafe file handling.
- Sensitive data exposure.

## Disclaimer

PokeVault TCG is provided as-is, without warranties of any kind. The maintainers will make reasonable efforts to review and address reported security issues when possible.
